# 現在の制限

この一覧は、動作済み機能と将来構想を混同しないためのものです。

## AI・文字起こし

- 実LLM整文providerは未接続。現在のMockは決定論的な経路確認用。
- 要約AIは本運用未実装。UIは空状態または既存Mockデータを表示するだけ。
- 外部クラウドSTTは本運用未実装。
- 発言者識別、話者別議事録、専門用語の組織管理UIはない。
- Local Whisper利用には各PCへのモデルと`whisper-cli.exe`配置が必要。
- PC性能、マイク、話し方、騒音により精度と遅延が変わる。

## データ

- 会議履歴の本格永続化はない。
- ページ再読込後に進行中会議や会議詳細を復元しない。
- IndexedDBには未ack音声と整文レコードが残り得るが、会議単位の管理UIではない。
- 保存期間、削除、バックアップ、監査の団体運用は未設計。
- エクスポート、高度な検索、共有リンクはない。

## 利用者・配備

- 認証、権限管理、管理者画面がない。
- 複数利用者、組織、会議ごとの分離がない。
- Local Whisperは保守的な単一セッション既定で、同時会議管理がない。
- HTTPS構成、証明書管理、公開サーバー向けrate limit・CSRF等の防御がない。
- 現在のバックエンドをそのままLANやインターネットへ公開できない。
- Docker、サービス常駐、監視、バックアップ、災害復旧は未実装。

## UI・評価

- 新しいhome／meeting／meeting-detail UIは自動テスト済みだが、実ブラウザーでの最終レイアウト確認が必要。
- 新UIでの実マイク、権限拒否、デバイス切替確認が必要。
- 狭いウィンドウ、長文、下部バー、スクロール追従の実操作確認が必要。
- 長時間会議、連続再接続、大量発話、メモリ使用量の評価が必要。
- アクセシビリティはコード上の配慮があるが、支援技術による実地評価は未実施。

## 安全性の境界

- `.gitignore`と監査スクリプトは誤共有リスクを下げるが、秘密情報の完全検出を保証しない。
- 保護トークン検証は保守的な文字列規則で、意味の正しさを完全判定しない。
- 法令・団体規則への完全準拠を保証するものではない。
- 外部API、有料サービス、一般公開は事前合意と追加実装が必要。
